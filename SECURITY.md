# Security Policy

## Supported Versions

This project is actively maintained on the default branch.
Security fixes are applied to maintained branches at maintainer discretion.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

1. Use GitHub private vulnerability reporting if enabled.
2. If private reporting is unavailable, open an issue with limited detail and request a private contact channel.
3. Include impact, reproduction steps, and affected files/routes.

We will acknowledge reports as soon as possible and coordinate a fix and disclosure timeline.
